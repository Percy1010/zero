# Project Cases

Each case is written for reuse, not for flashy demos.

## Suggested structure

1. Background: objective, deadline, and input materials.
2. Steps: task breakdown with key prompts.
3. Result: deliverable, time spent, and quality delta.
4. Retrospective: what worked, what failed, what to change.

## Minimum acceptance bar

- Include at least one reusable prompt template.
- Clearly show where human intervention is required.
- List risk points (factual errors, copyright, privacy).
