# 热门工具

这一章解决的问题不是“工具越多越好”，而是“当前任务到底该先学哪一个工具”。内容会按真实使用路径来组织，而不是只堆工具名单。

## 本章主要内容

这部分不再按“写作类 / IDE 类”去堆工具，而是按“一个工具一条学习路径”来组织。每个工具都会拆成：

1. 工具总览
2. 基础操作
3. 高阶使用

你会在这一章里看到：

- Claude Code、OpenCode、OpenClaw 这类主路径工具的完整学习入口
- LangChain 这类工程框架的补充阅读入口
- 少量旧示例页，帮助你了解历史分类方式，但它们不再是推荐主路径
- 每个工具从“先认识，再上手，再进阶”的学习顺序

## 如何学习更合适

- 如果你第一次接触 AI 编程工具，先读一个工具的总览页，再进入基础操作，不要多个工具同时开学
- 如果你已经在工作里有明确场景，优先选最接近当前任务的工具，学完基础操作后立刻上手真实任务
- 如果你只是想建立判断力，至少比较 2 个工具的定位差异，再决定后续长期使用哪一个
