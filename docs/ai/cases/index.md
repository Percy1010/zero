# 项目案例

这一章聚焦真实项目，不追求炫技案例，而是强调“这个过程你能不能照着复用”。重点不是 AI 有多聪明，而是它在哪一步真正帮你省了时间、又在哪一步必须人工兜底。

## 本章主要内容

每个案例都围绕“可复用”的目标来写，优先强调过程可复制。你会重点看到：

- 任务背景是什么，为什么适合引入 AI
- 具体怎么拆步骤，哪些 Prompt 真正起作用
- 结果有没有节省时间，质量有没有变化
- 风险和返工主要出现在什么环节

## 建议结构

1. 背景：任务目标、时限、输入材料。
2. 步骤：拆解任务并标出关键 Prompt。
3. 结果：交付物、耗时、质量变化。
4. 复盘：哪里有效、哪里失效、下次怎么改。

## 最低验收标准

- 给出至少 1 个可直接复用的 Prompt 模板。
- 说明“人工介入”发生在哪一步。
- 明确风险点（事实错误、版权、隐私）。

## 如何学习更合适

- 不要把案例当成结果展示，重点看中间过程怎么拆
- 读案例时优先记录“可直接复用的步骤”和“必须人工判断的节点”
- 最好挑一个和自己工作最接近的场景，边读边复做一遍

## 下一步

优先补齐 3 类高频场景：文档写作、信息整理、代码辅助。

## 学完后你应该获得什么

- 知道哪些任务适合让 AI 先做，哪些任务必须人工把关
- 能从真实案例中抽取出自己的可复用工作流
- 能更早识别 AI 项目落地时最常见的风险点
