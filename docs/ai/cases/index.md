# 项目案例

每个案例都围绕“可复用”的目标来写，不追求炫技，优先强调过程可复制。

## 建议结构

1. 背景：任务目标、时限、输入材料。
2. 步骤：拆解任务并标出关键 Prompt。
3. 结果：交付物、耗时、质量变化。
4. 复盘：哪里有效、哪里失效、下次怎么改。

## 最低验收标准

- 给出至少 1 个可直接复用的 Prompt 模板。
- 说明“人工介入”发生在哪一步。
- 明确风险点（事实错误、版权、隐私）。

## 下一步

优先补齐 3 类高频场景：文档写作、信息整理、代码辅助。
