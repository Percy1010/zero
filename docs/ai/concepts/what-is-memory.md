# 记忆(Memory)

记忆的英文是 `Memory`。在 AI 产品里，你可以把它理解成：**系统为了后续任务，把一部分信息保留下来并在将来继续使用的能力**。

## Memory 是什么

上下文像你桌面上临时摊开的资料，记忆像你贴在本子里的长期便签。

## 为什么用户会觉得“它记住我了”

当系统把这些内容保存下来并再次使用时，你就会感觉它“记得你”了，例如：

- 你偏好的输出语言
- 你的工作身份
- 你常用的文件结构
- 你之前已经确认过的规则

## 记忆和上下文有什么区别

- 上下文：当前这一轮给模型看的内容
- 记忆：被系统保存下来、以后还能继续调用的信息

## 记忆一般分哪几类

常见会分成：

- 短期记忆：围绕当前任务临时保留的信息
- 长期记忆：跨多次任务保留的偏好、规则和稳定事实

## 为什么不是所有东西都应该被记住

因为记忆一旦混乱，系统会越来越不稳定：

- 旧规则可能和新规则冲突
- 临时要求可能被错误地长期保留
- 记错的信息会反复影响之后的回答

## 哪些信息更适合被记住

- 长期偏好
- 项目规则
- 固定术语
- 稳定身份信息

## 你现在只需要记住什么

- 记忆不是普通聊天记录的同义词
- 记忆解决的是“以后还要不要继续用这些信息”
- 不是记得越多越好，而是记得越准越好
