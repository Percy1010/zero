# 上下文(Context)

## 一句话定义

`Context` 就是模型在当前这一次回答里能看到、能参考、能利用的全部信息。

## 通俗理解

如果你只说「我头疼」，医生很难判断问题。  
但如果你补充更多背景信息，判断质量就会显著提升。对模型来说，这些补充信息就是 Context。

## Context 里通常包含什么

- System Prompt
- 之前几轮对话
- 上传的文件
- 检索回来的资料
- 工具调用结果
- 用户当前的提问

## 它和 Prompt、Memory 的区别

- `Prompt`: 更像任务说明书
- `Context`: 当前这一轮模型能看到的全部信息
- `Memory`: 系统保存下来、以后还能继续调出来的信息

## 你现在只需要记住什么

- Context 是模型当前真正看到的信息
- 很多回答质量问题，本质上是 Context 问题
- 设计好 Context，往往比单纯换一种问法更重要
