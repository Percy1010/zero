# 上下文(Context)

上下文的英文是 `Context`。在 AI 里，你可以先把它理解成：**模型当前这一次能看到、能参考的信息集合**。

## Context 是什么

模型回答问题时，不是凭空作答，而是基于“当前看见的内容”来生成答案，这部分“当前看见的内容”就是上下文。

## 一个更好懂的类比

你可以把上下文想成桌面上摊开的资料：

- 桌面上有的，模型更容易参考
- 桌面上没有的，模型就只能靠自己已有的模式去猜

## 上下文通常包括什么

- 你的当前提问
- 之前几轮对话
- 系统说明
- 上传的文件
- 检索到的资料
- 工具调用返回的结果

## 为什么上下文重要

因为模型的输出质量，往往直接取决于上下文质量：

- 上下文不完整，答案就容易缺东西
- 上下文噪声太多，答案就容易跑偏
- 上下文层次不清，答案就容易抓不住重点

## 上下文和 Prompt 有什么区别

- Prompt 更像“你给的任务说明”
- Context 更像“模型当前所有可参考的信息”

Prompt 属于上下文的一部分，但上下文比 Prompt 更大。

## 上下文和记忆有什么区别

- 上下文是“当前这一轮模型看见了什么”
- 记忆是“系统是否把过去的重要信息保存下来，并在以后继续使用”

## 常见误区

### 误区一：上下文越长越好

不一定。太长的上下文会增加成本，也会把重点冲淡。

### 误区二：模型知道我之前想过什么

模型只知道它实际收到的内容，不会自动知道你脑子里的背景。

## 你现在只需要记住什么

- 上下文就是模型当前真正看见的信息
- 很多回答质量问题，本质上是上下文问题
- 给模型补对资料，往往比单纯换一种问法更有效
