# 大语言模型(LLM)

大语言模型是 `Large Language Model` 的中文说法，常写成 `LLM`。你可以先把它理解成：**一种擅长理解和生成语言的大型 AI 模型**。

## LLM 是什么

它最像一个“超大规模语言补全系统”，能够根据你给出的内容，预测接下来最合适的词、句子和结构。

## 用什么类比最容易懂

你可以把大语言模型想成一个读过海量文本的人：

- 它读过非常多公开语料
- 它擅长总结表达规律
- 它能根据你的问题，快速组织出一段像样的回答
- 但它不是把原文一字一句背出来，而是在“按概率生成”

## LLM 为什么会显得像“会思考”

从外部看，它像在思考；但从底层机制上看，它首先是在做“基于上下文的预测”。

## LLM 能做什么

- 写作和改写
- 总结和提炼
- 翻译
- 问答
- 代码解释和生成
- 把非结构化内容整理成更结构化的输出

## LLM 不能天然保证什么

- 不天然保证事实正确
- 不天然知道最新信息
- 不天然等于能访问你的文件、网页和数据库

## 它和搜索引擎有什么区别

- 搜索引擎擅长“找到已有内容”
- 大语言模型擅长“组织和生成新的回答”

## 它和数据库有什么区别

- 数据库存的是结构化、确定的信息
- 大语言模型存的是“参数化后的模式”

所以你不能把模型当数据库用。问模型一个冷门事实，它可能回答得很像真话，但并不代表这个事实真的被可靠保存。

## 为什么这个概念重要

因为很多后续概念都建立在它之上：

- Prompt 是你和模型交流的方式
- Context 是模型当前能看到的信息
- RAG 是给模型临时补资料
- Function Calling 是让模型请求外部工具
- MCP 是让模型接入外部能力的更通用协议

## 常见误区

### 误区一：参数越多就一定越强

参数规模很重要，但不是唯一因素。训练数据、训练方法、对齐方式和工具系统都会影响实际效果。

### 误区二：模型会说话，就一定真的懂世界

它可能很像懂，但它的强项首先是语言模式，不是现实世界的直接感知。

## 你现在只需要记住什么

- LLM 是 AI 系统里的“语言大脑”
- 它擅长生成和组织语言
- 它不等于搜索引擎，也不等于数据库
