# 检索增强生成(RAG)

RAG 是 `Retrieval-Augmented Generation` 的缩写，中文通常叫“检索增强生成”。

## RAG 是什么

RAG 可以理解成：**先找资料，再让模型基于这些资料生成回答**。

## 用什么类比最容易懂

你可以把它想成“开卷答题”：

- 先去翻资料
- 再根据资料组织答案

相比之下，单纯靠模型自己回答，更像“闭卷作答”。

## 为什么会需要 RAG

因为单靠模型本身有三个现实问题：

1. 它不一定知道最新信息
2. 它不一定知道你的私有资料
3. 它可能会把本来应该查证的内容直接猜出来

## RAG 的基本流程

一个最典型的 RAG 流程通常是：

1. 用户提问
2. 系统根据问题去资料库里检索
3. 把检索到的片段放进上下文
4. 模型基于这些片段生成答案

## RAG 和搜索有什么区别

- 搜索更像“先把资料找到”
- RAG 更像“找到资料后再交给模型整合回答”

## RAG 能解决什么问题

- 给模型补最新资料
- 给模型补私有知识
- 降低纯靠记忆回答时的幻觉风险
- 让答案更贴近具体场景和资料来源

## RAG 不能天然解决什么问题

- 检索错了，后面生成也会错
- 检索片段太多或太乱，模型仍可能抓错重点
- 如果资料本身有误，模型也会把错误组织得很像对的

## 你现在只需要记住什么

- RAG 的本质是“先检索，再生成”
- 它常用于知识库、客服、问答和文档助手
- 它不是替代模型，而是给模型补资料
