# 智能体(Agent)

`Agent` 通常翻译成“智能体”。你可以先把它理解成：**一个不仅会回答问题，还会根据任务目标自己决定下一步该做什么的 AI 执行者**。

## Agent 是什么

普通聊天模型更像“你问一句，它答一句”。  
Agent 更像“你交给它一个目标，它会自己拆步骤、找资料、调用工具，并逐步推进任务”。

## 用什么类比最容易懂

你可以把普通聊天 AI 想成前台问答员，把 Agent 想成项目助理：

- 前台问答员：只回答你当下的问题
- 项目助理：会根据目标主动安排步骤，必要时查资料、开工具、整理结果

所以 Agent 的关键不是“更会聊天”，而是“更会执行”。

## Agent 和工作流有什么关系

这两个概念经常挨着讲，但不一样：

- 工作流：步骤通常是人先设计好的
- Agent：会根据情况动态决定下一步

你可以先把工作流理解成固定流程，再把 Agent 理解成“能在流程里做更多自主决策的执行者”。

## 一个简单例子

如果目标是“帮我整理一份竞品分析”：

- 工作流方式可能是固定五步：搜资料、提取要点、分类、写总结、排版
- Agent 方式则可能在执行中自己判断：
  - 资料不够，先补搜
  - 资料太杂，先筛选
  - 缺少结构，先列提纲

也就是说，Agent 更像会自己调整路线。

## Agent 为什么需要工具

因为模型本身主要擅长生成文字，不天然擅长真正执行外部动作。

所以一个有用的 Agent 往往要能：

- 搜索
- 读文件
- 调接口
- 写入结果
- 调用其他子能力

这也是为什么在 Agent 后面，理解 MCP 和 Function Calling 会更自然。

## Agent 和聊天机器人有什么区别

- 聊天机器人更偏对话
- Agent 更偏完成任务

当然很多产品两者都会有，但从概念上区分，Agent 更强调“为了目标持续推进”。

## 常见误区

### 误区一：只要能聊天就是 Agent

不是。关键不在能不能聊，而在它会不会围绕目标持续行动。

### 误区二：Agent 一定比普通工作流更高级

不一定。对很多简单稳定任务，固定工作流反而更可靠。

### 误区三：Agent 越自主越好

也不一定。自主性越强，通常越需要更严格的权限、边界和验收标准。

## 你现在只需要记住什么

- Agent 是“会围绕目标持续推进任务”的 AI 形式
- 它比普通聊天更强调执行
- 理解 Agent 之后，再看 MCP 和 Function Calling 会更顺
