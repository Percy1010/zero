# 大语言模型(LLM)

## 一句话定义

`LLM` 是 `Large Language Model` 的缩写，中文叫「大语言模型」。你可以先把它理解成一个超大规模的文字预测机器: 你给它一段文字，它会根据前面的内容去预测下一个最可能出现的词或字。

## 通俗理解

### 类比 1: 超级学霸做完形填空

想象一个人读过海量的书、文章、代码、论文和帖子。现在你给他一句话的开头，让他继续往下写。他之所以经常能接得很像样，不是因为他把每篇文章都背下来了，而是因为他见过太多语言模式，知道什么样的词通常会接在什么样的词后面。

这就是 LLM 最核心的能力: 它先是「会接龙」，然后因为接龙能力极强，表现得像是在理解和思考。

### 类比 2: 被关在小黑屋里的天才

你只能通过门缝给这个天才递纸条，他看完后再写纸条回给你。

- 他读过非常多的资料
- 他很会组织语言
- 但他看不到外面的实时世界
- 也不会自己去操作工具
- 每次你不把背景交代清楚，他就只能靠猜

所以 LLM 的强项是语言生成，不是天然拥有现实世界的实时感知能力。

## 它的原理是什么

### 第 1 步: 训练

训练阶段可以先理解成「疯狂读书 + 不断做预测题」。

研究人员把海量文本喂给模型，让模型反复做同一件事: 根据前面的内容，猜后面最可能出现什么。猜错了就调整参数，猜对了就强化相关模式。经过极大量训练后，模型内部参数会逐步形成对语言、语气、结构、知识模式的压缩表示。

### 第 2 步: 生成

当你真正问它问题时，它不是一次把整段答案想完，而是一个 Token 一个 Token 往后生成。

例如:

```text
法国的首都是
```

模型会比较各种可能，然后逐步生成:

```text
法国的首都是巴黎
```

### 第 3 步: 对齐

原始训练出来的模型可能会乱说、有害输出、或者不按人类想要的方式回答。所以后面通常还会做一层「对齐」，让模型更符合人类偏好的回答风格和安全要求。

你可以把它理解成「上岗培训」: 不只是会做题，还要学会怎么更适合和人一起工作。

## 它能做什么

- 对话聊天
- 写作和改写
- 总结长文
- 翻译
- 写代码和解释代码
- 数据分析辅助
- 生成提纲、方案、脚本和文案

## 它不能天然做什么

- 不能自动联网获取最新信息
- 不能保证每次都说真话
- 不能天然访问你的数据库、文件和业务系统
- 不能跨对话长期记住你，除非系统额外做了 Memory

这些限制正是后面 `Search`、`RAG`、`Memory`、`Agent`、`MCP` 等概念出现的原因。

## 一个最小案例

### 案例 1: 写商务邮件

```text
我是一家设计公司的项目经理，需要给客户写一封邮件，
告知他们项目时间可能延期两周，原因是设计方案需要修改，
语气要诚恳专业，不能让客户生气。请帮我写。
```

模型通常就能写出一封结构完整、语气得体的邮件。

### 案例 2: 把它当老师

```text
我完全不懂会计，请用教小学生的方式，
帮我讲明白什么是借贷记账法，要有例子。
```

这类任务很适合 LLM，因为它擅长把复杂内容重新组织成更容易理解的语言。

## 为什么它是后续概念的起点

- `Prompt` 解决「怎么把任务说清楚」
- `Context` 解决「模型回答时看到了什么」
- `Memory` 解决「重要信息怎么延续」
- `Search` 解决「怎么拿到最新公开信息」
- `RAG` 解决「怎么拿到你的私有资料」
- `Function Calling` 解决「怎么让模型请求调用工具」
- `MCP` 解决「怎么把外部工具统一接进来」

## 你现在只需要记住什么

- LLM 先是一个强大的文字预测系统
- 它因为预测能力极强，才表现得像会理解和思考
- 它擅长语言，不等于它天然会联网、会记忆、会操作工具
