# 提示词(Prompt)

## 一、是什么？

### 一句话定义

提示词 `Prompt`，就是你输入给 AI 模型的指令、问题、背景、限制条件和格式要求。  
提示词工程 `Prompt Engineering`，则是研究如何构造和迭代 Prompt，让模型输出更稳定、更符合需求的一套方法。

## 二、工作原理

想象你在给同事布置任务。如果你只说一句「写个方案」，结果大概率不会理想。  
但如果你同时讲清楚:

- 做给谁看
- 要解决什么问题
- 采用什么语气
- 输出成什么格式
- 有哪些边界不能碰

对方完成任务的效果就会明显更好。

使用 LLM 也是一样。  
你的 Prompt 越清晰、具体、没有歧义，模型越容易给出符合预期的答案。

## 三、一个完整 Prompt 的组成结构

一个完整 Prompt 不一定每次都要写满，但理想情况下常包含这些部分:

```text
① 角色设定
   「你是一位资深的市场营销专家」

② 背景 / 上下文
   「我们公司是做母婴用品的，目标客户是 25-35 岁妈妈」

③ 任务指令
   「帮我写一段小红书推广文案」

④ 格式要求
   「字数 200 字以内，包含 3 个 emoji，带标签」

⑤ 示例
   「参考风格: 姐妹们！这个 XXX 真的绝了...」
```

## 四、核心技巧

### 1. 明确具体

不要说「写点关于狗的东西」，而应该说:

```text
请用生动活泼的语言，为 6-8 岁儿童写一段 100 字左右的关于金毛寻回犬性格特点的介绍。
```

### 2. 提供上下文

告诉模型你的身份、背景和目标。

### 3. 指定格式

如果你需要表格、分点、纪要、标题结构、长度限制，要明确写出来。

### 4. 使用进阶技巧

| 技巧名称 | 怎么做 | 适用场景 |
| --- | --- | --- |
| 角色扮演 | 「你是一位有 10 年经验的律师」 | 让回答更专业 |
| 思维链（CoT） | 「请一步步思考」 | 复杂推理、数学题 |
| 少样本提示 | 给 2-3 个示例 | 让 AI 模仿特定风格 |
| 提示链 | 把复杂任务拆成多步 | 长篇写作、复杂分析 |

## 五、通俗比喻

- Prompt 像点菜: 你说「来个菜」，厨师不知道做什么；你说清楚要求，结果才更准
- Prompt Engineering 像面试官的提问技巧: 好问题能把对方最好的能力引出来
- Prompt 像导航目的地: 地址越精确，导航越不容易带错

## 六、实战案例

### 差的 Prompt

```text
帮我写个文案
```

### 更好的 Prompt

```text
你是一位资深的母婴电商运营专家。我们即将上线一款婴儿湿巾新品，
主打天然有机、无添加、超柔软。请为我撰写一条小红书推广文案，要求:
1. 字数在 150-200 字之间
2. 语气亲切、口语化，像闺蜜分享好物
3. 包含至少 2 个 emoji
4. 结尾带 3 个相关标签
```

Prompt 工程本质上不是「写一句魔法咒语」，而是不断经历:

`编写 -> 测试 -> 分析 -> 迭代`

## 七、业务场景应用

| 业务场景 | Prompt 示例 |
| --- | --- |
| 客服标准回复 | 按「先道歉 -> 说明原因 -> 给出方案 -> 表达关怀」结构回复 |
| 周报生成 | 按「本周完成 / 进行中 / 下周计划」输出 |
| 会议纪要 | 提取参会人、议题、决议、待办、责任人、截止日期 |
| 数据分析 | 找出趋势，用非专业人士能懂的语言解释并给建议 |

## 八、你现在只需要记住什么

- Prompt 是任务说明书，不是神秘咒语
- Prompt Engineering 的核心是减少歧义、提升稳定性
- 不会写代码的人，也可以通过 Prompt 高质量地「驱动」大模型
