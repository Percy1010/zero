# 搜索(Search)

## 一句话定义

`Search` 就是让 AI 在回答前先去外部信息源里找资料，弥补模型知识有截止日期、也不一定知道当前具体细节的缺陷。

## 通俗理解

### 类比: 开卷考试

没有 Search 的模型更像闭卷考试，只能靠自己「脑子里记住的内容」答题。  
有了 Search 之后，它就变成了开卷考试: 碰到需要最新资料、具体事实、实时动态时，可以先去翻书。

## 它的工作原理

1. 用户提问
2. 系统判断这个问题是否需要搜索
3. 去网页、知识库、站内文档等地方找资料
4. 把结果交给模型阅读和整理
5. 模型基于结果输出回答

所以 Search 负责的是「找资料」，不是「最终表达」。

## Search 和 RAG 的区别

- `Search` 更常见于公开互联网或外部资料源
- `RAG` 更常见于你的私有知识库或受控资料库

## 你现在只需要记住什么

- Search 是给模型补外部资料的能力
- 它本身不等于最终答案
- 很多联网 AI，本质上是「先搜，再写」
