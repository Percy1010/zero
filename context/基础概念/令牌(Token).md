# 令牌(Token)

## 一、是什么？

### 一句话定义

`Token` 是 AI 大模型处理文本的最小单位，也是计费的基本单位。它不等于一个字，也不等于一个词，而更像一种「语义碎片」。

## 二、工作原理

AI 不能直接「读」文字，它需要先把文本转换成可计算的数字序列。这个过程叫 `Tokenization`。

例如:

```text
你输入: 「我爱北京天安门」

分词器可能处理成:
「我」 -> Token ID
「爱」 -> Token ID
「北京」 -> Token ID
「天安门」 -> Token ID
```

模型真正看到的是一串数字，而不是我们平时看到的自然语言。

## 三、Token 与计费

- 中文: 1 个汉字通常约等于 1.5-2 个 Token
- 英文: 1 个单词通常约等于 1-3 个 Token
- 标点、空格、数字也可能占 Token

大多数 API 都会对输入和输出分别计费。

## 四、为什么长对话会特别贵？

因为多轮对话时，系统往往会把前文历史一起重新发送给模型。

```text
第 1 轮: 提问 + 回答
第 2 轮: 第 1 轮历史 + 新提问 + 新回答
第 3 轮: 前 2 轮历史 + 新提问 + 新回答
```

所以 Token 消耗不是线性增长，而常常会越聊越快。

这也是为什么 `Memory` 压缩、`RAG` 精准检索、`Workflow` 拆分流程都很重要: 本质上都在省 Token。

## 五、通俗比喻

- Token 像拼图碎片
- Token 像出租车计价器
- 上下文窗口像油箱

## 六、实用建议: 如何省 Token？

| 方法 | 具体做法 |
| --- | --- |
| 精简提问 | 去掉废话，直奔主题 |
| 指定回答长度 | 如「用 100 字以内回答」 |
| 适时开新对话 | 历史太长时不要硬拖 |
| 使用记忆压缩 | 先总结前文，而不是全文搬运 |
| 按需使用模型 | 简单问题用便宜模型，复杂问题用贵模型 |

## 七、你现在只需要记住什么

- Token 是模型处理文本和计费的基本单位
- 成本和上下文窗口都与 Token 直接相关
- 很多工程优化，本质上都是在「让有限的 Token 更值钱」
