# 记忆(Memory)

## 一句话定义

`Memory` 是让原本每次都像「第一次见你」的模型，能够把重要信息保存下来，并在后续继续使用的一套机制。

## 通俗理解

### 类比 1: 金鱼记忆和日记本

模型天生更像「金鱼记忆」: 你不把旧信息重新给它，它就不会自动记住。  
Memory 机制就像给这条金鱼配了一本日记本:

- 对话结束后记下来
- 下次开始前翻出来
- 再把相关内容塞回 Context

### 类比 2: 酒店前台的便签本

前台不可能记住所有客人，但她会记:

- 哪位客人不吃辣
- 哪位客人喜欢靠窗
- 哪位客人之前投诉过什么问题

下次客人来时，系统翻出便签，就能显得「记得你」。

## Memory 一般分哪几类

- 短期记忆
- 长期记忆
- 摘要记忆

## 它的工作原理

1. 把对话历史或关键信息存起来
2. 在下一次调用前，取出相关内容
3. 再把它们放进新的 Context

所以 Memory 的本质不是「模型脑子里记住了」，而是「系统在帮它做外部记忆管理」。

## 你现在就能怎么做

### 方法 1: 手动 Memory

```markdown
## 我的信息
- 职业: 电商运营
- 行业: 宠物食品
- 当前目标: 拓展小红书渠道
- 风格偏好: 简洁直接
```

### 方法 2: 用产品自带的记忆功能

很多 AI 产品已经支持跨对话记忆、项目知识、长期偏好保存等功能。

### 方法 3: 建一份个人 Context 文档

把固定信息整理成一份文档，需要时直接粘给模型。

## 你现在只需要记住什么

- Memory 不是普通聊天记录的同义词
- 它的本质是「把以后还要用的信息保存下来」
- 不是记得越多越好，而是记得越准越好
