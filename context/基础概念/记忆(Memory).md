# 记忆(Memory)

## 一、是什么？

### 一句话定义

`Memory` 是让 AI 「记住」你之前说过的话、你的偏好、以及过去发生的事情的一套机制。它是 Agent 从「健忘的一次性问答机」进化为「长情的个人助理」的关键。

## 二、工作原理: 为什么 AI 会「忘事」？

LLM 本身更像「一问一答」系统。  
每次你发消息给它，系统通常会把「之前的历史 + 你的新消息」一起重新发给模型。

所以多轮追问看起来像 AI 记住了你，但很多时候只是它把聊天记录重新读了一遍。  
一旦历史太长，超出上下文窗口，旧内容就会被截断，模型就会显得「忘了」。

## 三、记忆的分类

| 记忆类型 | 人脑类比 | AI 实现方式 | 特点 |
| --- | --- | --- | --- |
| 短期记忆（STM） | 你正在想的事 | 会话缓冲、滚动窗口 | 容量小、更新快、仅限当前对话 |
| 长期记忆（LTM） | 多年积累的经验 | 摘要记忆、数据库、知识图谱、向量库 | 容量大、跨对话、持久保存 |

## 四、三种常见的短期记忆策略

```text
策略 1: 缓冲记忆
  -> 把全部历史都塞进去
  -> 问题: 容易超出窗口

策略 2: 窗口记忆
  -> 只保留最近 N 轮
  -> 问题: 早期重要信息容易丢

策略 3: 摘要记忆
  -> 先总结旧对话，再保留最近几轮
  -> 优点: 省 Token，还能保留关键信息
```

## 五、长期记忆的工作流程

长期记忆更像一个外部记事本:

1. 在对话中识别值得保存的信息
2. 把它存入数据库或向量库
3. 在未来需要时再检索回来
4. 重新放进上下文供模型使用

例如:

```text
你: 「我对花生过敏，以后推荐菜品时注意」
系统识别到关键偏好 -> 存入长期记忆

三个月后:
你: 「帮我推荐几道下酒菜」
系统先查到「花生过敏」 -> 自动避开相关菜品
```

## 六、通俗比喻

| 比喻 | 对应 |
| --- | --- |
| 便利贴 | 短期记忆，随手记，容易丢 |
| 笔记本 / 日记 | 长期记忆，可以回查 |
| 大脑 vs 外部笔记本 | 上下文窗口像大脑，长期记忆像外部存储 |
| RAM vs 硬盘 | 短期快但小，长期慢但大 |

## 七、业务场景应用

| 场景 | 记忆用法 |
| --- | --- |
| 智能客服 | 记住订单历史、投诉记录、偏好设置 |
| 个人助理 | 记住日程、家人信息、饮食偏好 |
| 教育 AI | 记住薄弱知识点、学习进度、错题 |
| 销售 CRM | 记住客户沟通历史、购买意向、决策链路 |
| 健康管理 | 记住过敏史、用药记录、健康目标 |

## 八、你现在只需要记住什么

- Memory 不是聊天记录的简单别名
- 它的价值在于「让重要信息跨轮次、跨会话继续生效」
- 没有 Memory，很多 Agent 只能像短时健忘的问答机
