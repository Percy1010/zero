# 上下文(Context)

## 一句话定义

`Context` 就是模型在当前这一次回答里能看到、能参考、能利用的全部信息。

## 通俗理解

### 类比 1: 去看医生

如果你只说「我头疼」，医生很难判断问题。  
但如果你补充:

- 已经疼了三天
- 是右侧太阳穴
- 最近连续加班、睡眠不好
- 去年做过检查
- 有偏头痛家族史

这些背景信息会显著提升判断质量。对模型来说，这些补充信息就是 Context。

### 类比 2: 电视剧的前情提要

「Previously on...」那段前情提要，就是帮观众快速进入状态的 Context。

## Context 里通常包含什么

- System Prompt
- 之前几轮对话
- 上传的文件
- 检索回来的资料
- 工具调用结果
- 用户当前的提问

## 它和 Prompt、Memory 的区别

- `Prompt`: 更像任务说明书
- `Context`: 当前这一轮模型能看到的全部信息
- `Memory`: 系统保存下来、以后还能继续调出来的信息

## 关键限制: 上下文窗口

每个模型一次能处理的 Context 都有上限，这个上限叫「上下文窗口」。

你可以把它理解成会议桌上的座位数:

- 信息越多，占的座位越多
- 座位满了，就必须删掉一部分
- 塞太多资料，不只更贵，也更容易让重点丢失

## 怎么做 Context Engineering

1. 只给相关信息
2. 用标题、表格、列表把信息结构化
3. 把最重要的信息放在前面或结尾
4. 明确区分「背景」和「任务」

## 你现在只需要记住什么

- Context 是模型当前真正看到的信息
- 很多回答质量问题，本质上是 Context 问题
- 设计好 Context，往往比单纯换一种问法更重要
