# 智能体(Agent)

## 一句话定义

`Agent` 就是一个围绕目标持续推进任务、还能调用工具的 AI 执行者。它让 AI 从「只会说话」变成「能干活」。

## 通俗理解

### 类比 1: 超级私人助理

模型像一个很聪明的顾问，但它自己不能真正去查票、发邮件、翻数据库、改文件。  
Agent 就像这个顾问旁边的私人助理:

- 负责转达你的需求
- 在需要时去调用工具
- 把结果带回来给模型继续判断
- 最后把任务完成情况告诉你

### 类比 2: 乐高机器人

```text
Agent = 大脑(LLM) + 手脚(工具) + 控制程序(循环逻辑)
```

## 它的核心原理: ReAct 循环

1. 思考
2. 行动
3. 观察
4. 再思考

这个过程会不断重复，直到任务完成。

## 它和 Workflow 的区别

- `Workflow` 更像固定流程
- `Agent` 更像在流程里自己判断下一步

## 你现在只需要记住什么

- Agent 的关键不是更会聊天，而是更会执行
- 它的本质是「围绕目标持续推进任务」
- 理解 Agent 之后，再看 `Function Calling` 和 `MCP` 会顺很多
