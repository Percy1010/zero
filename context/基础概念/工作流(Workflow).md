# 工作流(Workflow)

## 一句话定义

`Workflow` 是把一个复杂任务拆成多个明确步骤，并按固定顺序执行的一种组织方式。

## 通俗理解

### 类比 1: 可执行的流程图

你在工作里画过流程图:

1. 接收需求
2. 收集资料
3. 整理信息
4. 生成初稿
5. 审核并输出

Workflow 的意思，就是把这张流程图真正变成能跑起来的程序。

### 类比 2: 标准作业流程

Workflow 就是 AI 时代的 SOP: 任务不是想到哪做到哪，而是按步骤推进。

## 它和 Agent 的区别

- `Workflow`: 步骤通常预先设计好
- `Agent`: 运行时可以根据情况自己决定下一步

## 你现在只需要记住什么

- Workflow 的本质是「把复杂任务拆成稳定步骤」
- 好用的 AI 系统，往往依赖流程设计，而不是只靠一句神奇 Prompt
- 在很多场景里，固定 Workflow 比全自动 Agent 更可靠
