# 检索增强生成(RAG)

## 一句话定义

`RAG` 是 `Retrieval-Augmented Generation` 的缩写，中文通常叫「检索增强生成」。它的核心做法是: 回答前先查你指定的资料库，再基于查到的内容来生成答案。

## 通俗理解

### 类比 1: 带小抄的考试

你脑子里可能记得一些知识，但不一定准确。  
如果考试允许你带一本自己整理的小册子，你就会:

1. 先看题目
2. 再翻资料
3. 找到相关内容
4. 基于资料作答

### 类比 2: 配了图书馆的教授

没有 RAG 的模型，像一个博学但可能会凭印象回答的教授。  
有了 RAG 之后，它像一个旁边有专属图书馆的教授: 回答前先去查资料，再基于资料发言。

## RAG 的核心流程

### 阶段 1: 准备知识库

1. 导入文档
2. 切片
3. 向量化
4. 存入向量数据库

### 阶段 2: 用户提问时检索

1. 把问题转成向量
2. 找最相近的片段
3. 把这些片段塞进 Context
4. 让模型基于它们生成回答

## 什么是「向量」

可以先把向量理解成「文字的语义指纹」。

## 你现在只需要记住什么

- RAG 的核心是「先检索，再生成」
- 它是给模型补私有知识和可靠资料的常见方案
- 它不能替代模型，但能显著降低纯靠猜的风险
