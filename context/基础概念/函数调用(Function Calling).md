# 函数调用(Function Calling)

## 一句话定义

`Function Calling` 是模型和宿主程序之间的一种格式约定: 当模型觉得需要调用工具时，不是用自然语言含糊地说，而是按结构化格式把「调用哪个工具、参数是什么」表达出来。

## 通俗理解

### 类比 1: 标准化点菜单

如果餐厅服务员是机器人，你不能说「来个辣一点的鸡肉菜」，因为它很难精确理解。你必须填表。

### 类比 2: 去银行填转账单

你不能只说「帮我转点钱给我妈」，而要按字段填写。

## 它的工作原理

1. 开发者先告诉模型有哪些可用函数
2. 模型判断当前任务需不需要调用工具
3. 如果需要，模型输出结构化调用请求
4. 宿主程序解析这个请求，真的去执行函数
5. 执行结果再回给模型
6. 模型基于真实结果组织最终回答

关键点在于: 模型只是「请求调用」，真正执行的是外部程序。

## 它和 MCP 的区别

- `Function Calling`: 解决「模型怎么表达我要调哪个工具」
- `MCP`: 解决「工具和数据源怎么以统一协议接进系统」

## 你现在只需要记住什么

- Function Calling 让模型从「会说」开始走向「会做」
- 模型不直接执行函数，程序才真正执行
- 结构化格式是它可靠工作的关键
